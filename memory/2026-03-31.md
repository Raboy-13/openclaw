### Lessons learned

**MRO / mixin method resolution issues:**
- When splitting a class into mixins, `@staticmethod` and `@classmethod` methods lose their decorator
- Without the decorator, `self.method()` passes `self` as first arg → "takes 1 positional argument but 2 were given"
- Always check for `@staticmethod` and `@classmethod` before splitting mixins and add `self`/`cls` to the function signature

**Cross-mixin method calls:**
- When Mixin A calls `self.method_defined_in_mixin_B`, it works at runtime via MRO
- But if method resolution order puts A before B in inheritance chain and A's method is called before B's method is defined → AttributeError
- Always group called methods together or move caller to the mixin that defines the callee

**Dynamic OCR crop formula:**
- `price_crop_y = abs_y - 5` placed the crop ABOVE the button (wrong!)
- Fixed by using pre-calibrated `price_ocr_region` constants from `constants.py` instead of dynamic offsets

**`sessions_spawn` with `runtime=acp`:**
- ACP pairing for Claude Code on Windows is broken — acpx can't maintain persistent sessions when spawned by the gateway (process group/PTY issue on Windows)
- `sessions_spawn` fails with "neither 'sessions ensure' nor 'sessions new' returned valid session identifiers"
- Direct `claude -p "..."` via shell works fine
